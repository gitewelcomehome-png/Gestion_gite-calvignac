3. Historique des corrections

07 mars 2026 — Identification du problème
Pipeline silencieusement cassé depuis 3 semaines
La fonction upsert_error_log retournée par pg_proc était un stub (BEGIN NULL; END). Chaque appel RPC depuis error-tracker.js réussissait (code 200) mais n'écrivait rien en base. La vue cm_error_logs était donc toujours vide — le dashboard admin-monitoring n'affichait jamais rien.
28 mars 2026 — Migration v1 (partielle)
Première tentative : ALTER TABLE sur VIEW
Première migration ciblant cm_error_logs avec ALTER TABLE — erreur : cm_error_logs est une VIEW, pas une TABLE. Colonnes incorrectes utilisées (message au lieu de error_message). Migration échouée.
29 mars 2026 — Migration v2 (exécutée en production)
Fix complet : stub → vraie fonction + VIEW reconstruite
Étape 1 : ALTER TABLE public.error_logs (TABLE réelle) — ajout de 5 colonnes manquantes : source, user_email, occurrence_count, last_occurrence, affected_users.

Étape 2 : DROP VIEW / CREATE VIEW public.cm_error_logs — reconstruction avec aliases corrects (19 colonnes).

Étape 3 : CREATE INDEX IF NOT EXISTS idx_error_logs_dedup sur la TABLE réelle.

Étape 4 : CREATE OR REPLACE FUNCTION upsert_error_log — logique INSERT/UPDATE avec dédup 24 h. Remplace définitivement le stub.

Fichier : sql/FIX_UPSERT_ERROR_LOG_2026-03-28.sql
29 mars 2026 — error-tracker.js
Retry borné + filtres IGNORED_PATTERNS allégés
Retry limité à 5 tentatives avec backoff exponentiel (1 s → 5 s). Suppression des patterns trop larges de la liste IGNORED_PATTERNS : 'column' et 'does not exist' masquaient les vraies erreurs SQL.
30 mars 2026 — Webhook + merge preprod→main
Webhook auto-trigger Cowork déployé en production
Création de api/webhooks/supabase-error.js : endpoint Vercel déclenché par Supabase sur INSERT dans error_logs. Fix critique : crypto.timingSafeEqual crashait (500) sur GET car les buffers avaient des tailles différentes — ajout d'une vérification bufA.length === bufB.length + try/catch global. Merge preprod→main effectué (commit b3e9c30). URL prod : https://liveownerunit.fr/api/webhooks/supabase-error.
